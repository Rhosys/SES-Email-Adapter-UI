# Implementation Plan: Site UI Gaps

## Overview

Eight frontend features for the email-catcher site: stats dashboard (widget + full view), forwarding verification, domain delete, webhook rule action, retention duration setting, post-send intent buttons, undo toast UX improvements, and always-visible "Available until" on arc detail.

## Notes

- The stats backend (`GET /accounts/:id/stats`) may not return daily/monthly data yet — the widget should gracefully handle an API that only returns totals (show donut + totals, empty area chart) until the backend TODO is implemented.
- The webhook test fires from the browser — CORS may block it on many endpoints. The UI should show the error clearly ("Request blocked — your endpoint may need CORS headers, or it will work when called from our servers").
- The "Send + Wait" followup at 7 days is hardcoded for now. A future iteration could let the user pick a duration.
- Domain delete cascades aliases on the backend — the frontend doesn't need to refresh aliases after domain delete (the aliases tab will re-fetch on next visit).
